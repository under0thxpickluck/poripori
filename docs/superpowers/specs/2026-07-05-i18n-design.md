# MIRAIX 全画面多言語化(i18n) 設計仕様書

作成: 2026-07-05 / ステータス: ユーザー承認済み設計（実装前）

## 1. 目的

英語を標準言語とする国際サービスの体裁に合わせ（居住国ゲート・英文正文の同意と整合）、
全画面を多言語化する。

## 2. 対応言語

| コード | 言語 | 位置づけ |
|---|---|---|
| `en` | English | **標準（デフォルト）** |
| `ja` | 日本語 | ソース言語（コード内の原文） |
| `zh` | 中文（简体） | 参考訳 |
| `ko` | 한국어 | 参考訳 |
| `es` | Español | 参考訳 |
| `pt` | Português | 参考訳 |

- 法的文言（居住国ゲートの同意10項）は**英語正文のまま翻訳しない**（既存方針）。
- 規約・FAQ等の長文は参考訳の位置づけ（英語・日本語以外はその旨を将来注記可能）。
- 言語は後から辞書ファイル追加のみで増やせる構造にする。

## 3. 方式（追加依存ゼロ）

- `src/lib/i18n.tsx`: 自前の軽量 i18n。
  - **キー = コード内の日本語原文**。`t('残高')` のように使う。
  - `ja` は辞書不要（キーをそのまま返す）。他言語は `Record<原文, 訳文>` の辞書。
  - 訳が無いキーは日本語原文にフォールバック（画面が壊れない）。
  - 変数は `{n}` プレースホルダ: `t('罠 {n} 個', { n: 3 })`。
  - `useT()` フック + `I18nProvider`。言語変更で全体再レンダー。
- 辞書: `src/locales/en.ts / zh.ts / ko.ts / es.ts / pt.ts`（named export `dict`）。
- 言語の決定順: `localStorage('miraix_lang')` → `navigator.language` の前方一致 → `en`。
- 切替UI: Navbar に地球儀アイコンのドロップダウン（全ページ共通・未ログインでも可）。

## 4. 対象範囲

全画面: Navbar / Footer / MobileTabBar / ホーム(MarketList) / MarketDetail /
Portfolio / Dashboard / Wallet / SalonLink / Mines / Plinko / GameCorner /
各モーダル(Login / DailyBonus / LevelUpToast / ResidencyGate の UI 部分) /
Faq / Terms / Help / admin ページ群。

対象外: 同意10項の英文（正文固定）、DevAccountSwitcher（開発専用）、
サーバー側エラーコード（mapRpcError の訳語は対象）。

## 5. 検証

- `npx vitest run` / `npm run build` / `npx tsc --noEmit`
- デモモードで en / zh 切替のスクリーンショット確認（ホーム・Mines・ゲート）
- 言語設定がリロード後も維持されること
