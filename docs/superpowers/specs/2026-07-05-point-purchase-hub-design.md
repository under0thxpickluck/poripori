# ポイント購入ハブ設計（EPウォレット画面の拡張）

- 日付: 2026-07-05
- 対象: `src/pages/Wallet.tsx`、`src/components/Navbar.tsx`、`src/pages/Portfolio.tsx`、`src/locales/*.ts`
- 種別: UI変更（バックエンド・転送ロジックは変更なし）

## 目的

現在の `/wallet`（EPウォレット）は、サロンEP ⇄ MIRAIXポイント（MR）の転送フォームのみの画面。
これを「**ポイント購入**」ハブに再構成し、複数の決済手段を並べる。BTC / USDT / MetaMask /
クレジットカードは「整備中」のダミー（押下無反応）とし、**LIFAI EP連携のみが機能**して既存の
転送フォームにつながる。「EPウォレットへの導線」を、決済手段の1つという文脈に置き換えるのが狙い。

## スコープ外（変更しないもの）

- `supabase/functions/ep-transfer/index.ts`（転送・残高照会・resumeのロジック）
- 出金制限（bonus_locked / 日次上限 migrate-014 / REGION_BLOCKED migrate-016）
- ルーティングパス `/wallet` 自体
- 転送レート（1 EP = 1 MR）や入力上限（1〜10,000）

## アクセス制限（採用案: B）

現行の「サロン連携済みのみ閲覧可」ゲートを**外す**。サインイン済みなら未連携でも購入ハブを表示する。

| 状態 | 挙動 |
|------|------|
| 未サインイン（`!profile`） | 現行どおり「サインインしてください」表示 |
| ローカルデモ（`IS_LOCAL`） | 購入ハブは表示。ただし全手段が利用不可（LIFAI EPも「デモモードでは利用できません」） |
| サインイン済み・未連携 | 購入ハブ表示。LIFAI EPを選ぶと「サロン連携が必要です」案内＋ホームへの導線 |
| サインイン済み・連携済み | 購入ハブ表示。LIFAI EPを選ぶと既存の転送UI一式が展開 |

## 画面構成（`Wallet.tsx`、上から）

1. **ヘッダー**: タイトルを「EPウォレット」→「ポイント購入」に変更。サブに現在のMR残高を常時表示。
2. **新規登録特典バナー**: 既存のまま（`welcome` クエリ時のみ）。
3. **購入方法セレクター**（新規）: 手段カードを縦並び。
4. **選択中手段の詳細エリア**: 選択された手段に応じて下に展開。

### 購入方法セレクター

各手段はカード（アイコン＋ラベル＋状態バッジ）。

| 手段 | アイコン | 状態 | 押下時 |
|------|----------|------|--------|
| BTC | ₿（lucide `Bitcoin`） | 整備中バッジ・淡色・`cursor-not-allowed` | onClickなし（無反応） |
| USDT | 💵（lucide `CircleDollarSign` 等） | 整備中 | 無反応 |
| MetaMask | 🦊（lucide `Wallet` 等） | 整備中 | 無反応 |
| クレジットカード | 💳（lucide `CreditCard`） | 整備中 | 無反応 |
| LIFAI EP連携 | ✨（lucide `Sparkles`／既存 `ArrowLeftRight`） | 選択可能 | 選択状態にし詳細を展開 |

- 選択中の手段はハイライト（`border-accent bg-accent/10`）。
- 整備中カードは選択状態を持たない（状態変数に入らない）。純粋な飾り。
- 選択状態は `useState<'lifai' | null>`（整備中は選べないので実質2値）。初期値 `null`。

### LIFAI EP を選択したときの詳細（`selected === 'lifai'`）

`linked && !IS_LOCAL` の場合、既存UIをそのまま展開:
- 外部サイト免責バナー（既存）
- サロンEP / MR 残高2カード（既存、`loadEpBalance` 結果）
- 方向トグル（サロン→MIRAIX / MIRAIX→サロン）＋金額入力＋クイック＋転送ボタン（既存 `transfer()`）
- 転送履歴＋再開ボタン（既存 `resume()`）

`!linked`（連携なし）の場合:
- 「サロン連携が必要です。LIFAI Arcade から MIRAIX を開いてください」案内＋ホームへのリンク
  （現行の未連携メッセージを、この詳細エリア内に移設）

`IS_LOCAL` の場合:
- 「ローカルデモモードではLIFAI EP連携は利用できません」表示

### データ取得の条件

`loadHistory` / `loadEpBalance` / `loadDailyQuota` は現行どおり **`linked && !IS_LOCAL` のときのみ**
実行（`useEffect` のガード維持）。未連携ユーザーではサロン系APIを一切呼ばない。

## 導線の変更

### `Navbar.tsx`

- 現行: `salonLinked && <Link to="/wallet">EPウォレット</Link>`
- 変更後: `user && <Link to="/wallet">ポイント購入</Link>`（サインイン済み全員に表示、ラベル変更）

### `Portfolio.tsx`

- 現行: `salonProfile?.salon_login_id && <Link to="/wallet">…EPウォレット…</Link>`
- 変更後: `user && <Link to="/wallet">…ポイント購入…</Link>`
  - カード説明文を「サロンEP ⇄ MIRAIXポイントの転送・履歴」→「暗号通貨・LIFAI EP連携でポイントを購入」に更新。

（`MobileTabBar` は変更なし。ポートフォリオ経由の入口を維持。）

## i18n

新規文言はすべて `t()` 経由とし、`src/locales/en.ts`（EN標準）を基準に
`ja` を含む各ロケール（en/ja/ko/zh/es/pt）へ追加する。主な新規キー:

- 「ポイント購入」
- 「購入方法」
- 「整備中」
- 「クレジットカード」
- 「LIFAI EP連携」
- 「サロンEP（LIFAI）をMRに移して購入」等の説明文
- 「暗号通貨・LIFAI EP連携でポイントを購入」（ポートフォリオカード説明）
- 「ローカルデモモードではLIFAI EP連携は利用できません」

既存キー（免責・残高ラベル・転送フォーム・履歴・エラー文言）は再利用。

## テスト / 検証観点

自動テストは無い前提。手動で以下を確認:

1. **未連携ユーザー**: `/wallet` が表示され、整備中4手段は無反応、LIFAI EP選択で「連携が必要」案内。
2. **連携済みユーザー**: LIFAI EP選択で残高・転送フォーム・履歴が展開し、転送が従来どおり動作。
3. **整備中ボタン**: クリックしても状態変化・遷移・エラーが一切起きない。
4. **導線**: Navbar / Portfolio の入口がサインイン済み全員に「ポイント購入」で表示され `/wallet` に遷移。
5. **ローカルデモ**: 購入ハブは見えるが全手段利用不可の表示。
6. **i18n**: 言語切替で新規文言が各言語で表示され、未翻訳キーのフォールバックが崩れない。
